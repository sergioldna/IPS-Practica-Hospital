Contributing
=============

Thanks for your interest in contributing to this project. A few simple guidelines:

- Fork the repository and create a feature branch: `git checkout -b feat/your-change`
- Run tests locally before opening a PR:

```bash
./mvnw test
```

- Ensure integration tests pass when relevant:

```bash
./mvnw verify
```

- Follow existing code style and keep changes focused.
- Add unit/integration tests for bug fixes or new features.
- Open a pull request with a clear description of the change and the motivation.

For important design changes, open an issue first to discuss the approach.
